Up in 4 steps

From zero to automated standups in under 15 minutes.

1

Create your Slack App

Go to api.slack.com/apps → Create New App → paste the manifest from the repo. Install and copy your Bot Token and Signing Secret.

2

Configure your teams

Copy teams.yaml.example to teams.yaml. Add Slack user IDs, schedule, and timezone. One file controls everything.

3

Deploy with Helm

One command to your Kubernetes cluster. Helm handles deployment, service, ingress, secrets, and configmap.

✓

Standups run automatically

Every weekday at the configured time, Morgenruf DMs your team, collects answers, and posts formatted summaries. Done.

Deploy in one command

Self-hosting? Pick your method.

helm upgrade --install morgenruf   oci://ghcr.io/morgenruf/helm/morgenruf   --namespace morgenruf   --create-namespace   --set secret.slackBotToken="xoxb-..."   --set secret.slackSigningSecret="..."
docker run -d   -e SLACK_BOT_TOKEN=xoxb-...   -e SLACK_SIGNING_SECRET=...   -v ./teams.yaml:/app/teams.yaml   -p 3000:3000   ghcr.io/morgenruf/morgenruf:latest
git clone https://github.com/morgenruf/morgenruf
cd morgenruf/app
cp .env.example .env && cp teams.yaml.example teams.yaml
pip install -r requirements.txt
python src/main.py
